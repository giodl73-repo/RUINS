## What This Module Proves

RUINS tests whether lost, damaged, abandoned, erased, and fragmentary cultural
objects can be studied as a coherent evidence problem without reducing them to
travel lore, collapse romance, sensational hidden history, or generic
archaeology.

The central claim is methodological:

> A ruin is not just a surviving object. It is a structured relation between
> absence, trace, inference, reconstruction, and later memory.

RUINS should produce papers that make absence legible. Each paper should say
what is missing, what survives, what can be inferred, what is contested, what
later memory added, and what the researcher must refuse to claim.

## Core Vocabulary

| Term | Working meaning |
|---|---|
| Surviving trace | Material, textual, environmental, institutional, copied, translated, or reported residue. |
| Loss event or process | A bounded event or long process that damaged, dispersed, erased, abandoned, transformed, or decontextualized the original. |
| Evidence condition | The current state of the record and how directly it bears on a claim. |
| Reconstruction | A modern model, restoration, replica, translation, visualization, or operating hypothesis. |
| Cultural memory | Later symbolic, political, artistic, religious, commercial, or scholarly use of the ruin. |
| Refusal | A stated boundary around what the evidence does not support. |

## Publication Standard

Every RUINS publication should include:

1. research question,
2. core argument,
3. primary evidence condition,
4. source map,
5. chronology or loss-process table,
6. primary number or classification output,
7. null fallback,
8. claim-grade table,
9. reconstruction-status table,
10. cultural-memory afterlife table,
11. refusal section,
12. sibling Knowledge Systems handoffs,
13. reviewer notes.

## Non-Goals

- RUINS is not an ancient mysteries brand.
- RUINS is not a travel guide or site catalog.
- RUINS is not a looting, treasure-hunting, or site-location aid.
- RUINS does not assert suppressed certainty from fragmentary evidence.
- RUINS does not start with code, schemas, crates, or generated fixtures.
