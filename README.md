# RUINS - Partial Survival and Responsible Reconstruction

RUINS is a Knowledge Systems repo about partial survival: lost cities,
destroyed libraries, fragmentary texts, abandoned infrastructure, vanished
techniques, broken monuments, shipwrecks, erased settlements, and contested
reconstructions.

RUINS is paper-first. It does not start with code, schemas, crates, or generated
fixtures. Its first job is to define the research discipline: how to separate
surviving evidence, responsible inference, legend, modern projection, and
downstream cultural memory.

## Thesis

Most catalogs of ruins organize by place, period, or artifact type. RUINS
organizes by epistemic condition:

- what is gone,
- what survives,
- how surviving traces are interpreted,
- where claims become contested,
- what cannot be responsibly reconstructed,
- how the loss shaped later knowledge, identity, technology, or myth.

RUINS is not a travel guide, nostalgia archive, treasure-hunt index, or ancient
mysteries project. It is a knowledge system for handling absence carefully.

## Review Roles

RUINS uses repository-local review roles:

- Fragment Custodian,
- Absence Historian,
- Reconstruction Skeptic,
- Cultural Memory Reader,
- Sibling Boundary Editor,
- Paper Editor,
- Safety Editor.

## Method Evolution

RUINS follows the portfolio pattern: build the artifact, review it against an
explicit rubric, then let repeated findings improve the process.

- [`docs/rubric-amendments.md`](docs/rubric-amendments.md) defines the
  forward-only amendment policy.
- [`docs/capability-expansion.md`](docs/capability-expansion.md) tracks
  case-derived RUINS capabilities and repeated rubric gaps.
- [`docs/capability-promotion-packet.md`](docs/capability-promotion-packet.md)
  gives ASHLAR the fast-promotion checklist for new capabilities.
- [`docs/capability-roadmap.md`](docs/capability-roadmap.md) identifies the
  next papers and method artifacts that promote active capabilities to core.
- [`skills/README.md`](skills/README.md) operationalizes paper scaffolding,
  source locking, role review, and method evolution.

## Non-Goals

- No claims of secret history, suppressed proof, or sensational certainty.
- No treasure-hunting instructions or site-looting enablement.
- No real-time archaeological advice or conservation guidance.
- No fictional canon catalog unless clearly marked as downstream game/world
  design material.
- No implementation work until the paper method survives at least five unlike
  case studies.
