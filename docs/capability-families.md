# RUINS Capability Families

Status: active synthesis map
Date: 2026-07-08

RUINS capability families group Core capabilities by evidence pressure. A
capability can appear in more than one family, but the primary family should
control next-paper discovery, rubric gap review, and ASHLAR promotion pressure.

## How To Use

1. Pick a family with weak recent coverage or an unresolved overclaim pattern.
2. Choose a paper that adds a new evidence condition inside that family.
3. Promote only when the case creates a gate, register, cap, or sibling
   boundary that can be reused.
4. Update the family row when a new Core capability changes the method shape.
5. Use cross-family pressures to find synthesis papers instead of appending
   unrelated case studies.

## Promotion Review Questions

1. Which family owns the primary pressure?
2. Is the case adding a new evidence condition or repeating a settled cap?
3. Which sibling boundary is most likely to fail?
4. What exact overclaim should become an automatic cap?
5. Does the case add a reusable gate, register, checklist, or scorecard line?
6. What family gap remains after promotion?
