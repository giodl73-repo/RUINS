# Ancient Genomics Reconstruction Custody

**Capability id:** `ruins-capability:ancient-genomics-reconstruction-custody`
**Promotion:** Candidate -> Active -> Core
**Trigger papers:** `research/publications/panel-cheddar-man-ancient-genomics-custody/`; `research/publications/panel-kennewick-ancient-one-genomics-custody/`
**Date:** 2026-07-07

## Pressure

Ancient DNA can turn fragmentary remains into ancestry graphs, phenotype
predictions, facial reconstructions, population-replacement stories, and public
identity claims. RUINS needs a gate between genome data and biography, race,
continuity, or national origin.

## Activation Rule

Activate this capability when a paper reconstructs ancestry, phenotype,
population history, kinship, diet adaptation, appearance, or public identity
from ancient DNA, genomic capture, imputation, reference panels, or facial
reconstruction.

## Required Gate

Separate sample, contamination, coverage, authentication, reference panel,
model, phenotype marker, reconstruction choice, population-level inference,
individual identity claim, living-community relation, and public memory.

## Refusals

- Genome affinity is not modern race.
- Phenotype prediction is not exact face or identity.
- Population replacement is not total cultural replacement.
- A reconstructed face is not the person returned.
- Public ancestry stories are not final archaeological conclusions.

## Core Rule

Ancient genome data are reconstruction constraints, not direct identity. Sample,
authentication, coverage, reference panel, model, phenotype, reconstruction,
population inference, governance, and public memory must remain separately
inspectable before RUINS scores ancestry, appearance, affiliation, or population
history claims as strong.

## Decision

Promote to Core under rubric v1.36 with
