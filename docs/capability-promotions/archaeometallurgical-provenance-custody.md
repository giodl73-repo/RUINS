# Archaeometallurgical Provenance Custody

**Capability id:** `ruins-capability:archaeometallurgical-provenance-custody`
**Promotion:** Candidate -> Active -> Core
**Trigger paper:** `research/publications/panel-archaeometallurgical-provenance-custody/`
**Rubric:** v1.61

## Problem

Archaeometallurgical data can connect objects, slag, ores, and production debris
to possible source and technology histories. Those connections are fragile.
Slag chemistry, slag inclusions, metal compositions, lead-isotope ratios,
trace-element clusters, XRF/pXRF results, LA-ICP-MS datasets, SEM-EDS phases,
XRD mineral calls, metallographic structures, corrosion layers, ore database
matches, furnace fragments, crucible residues, and tuyere slags can be
overpromoted into exact mine sources, direct trade routes, certain workshops,
complete smelting recipes, production scale, political economy, ethnic or craft
identity, recycling history, technological superiority, or solved metal
circulation.

## Core Rule

Metallurgical provenance claims must keep sample, object/context, ore field,
analytical method, contamination/corrosion, recycling/mixing, isotope overlap,
reference database, chaîne opératoire, production stage, provenance
probability, exchange inference, and public industry memory separate.

## Required Controls

- Metallurgical-provenance gate.
- Source lock.
- Page-anchor audit.
- Refusal register.
- Cultural-memory register.
- Sibling-boundary register.
- Cross-case checklist.

## Decision

Promote directly to Core under rubric v1.61. Archaeometallurgical provenance
custody creates a reusable control for ore-source, slag, metal-composition,
lead-isotope, recycling, workshop, exchange, and ancient-industry claims.
