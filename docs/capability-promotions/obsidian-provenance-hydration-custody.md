# Obsidian Provenance-Hydration Custody

**Capability id:** `ruins-capability:obsidian-provenance-hydration-custody`
**Promotion:** Candidate -> Active -> Core
**Trigger paper:** `research/publications/panel-obsidian-provenance-hydration-custody/`
**Rubric:** v1.54

## Problem

Obsidian sourcing and hydration dating can connect artefacts to volcanic glass
sources and sometimes provide chronological pressure. Those signals are easily
overpromoted. Source assignments, XRF/pXRF clusters, INAA or LA-ICP-MS results,
hydration rims, diffusion rates, source distance, cortex, flake counts, and
reduction stages can be collapsed into direct trade routes, exact artefact
ages, migration paths, ethnic identity, territorial boundaries, complete
procurement systems, market networks, workshop organization, political
alliances, or whole interaction spheres.

## Core Rule

Obsidian claims must keep artefact, glass source, instrument method,
calibration, source assignment, sub-source ambiguity, hydration surface, rate
model, thermal history, context, reduction/discard process, exchange inference,
and public network memory separate.

## Required Controls

- Obsidian-provenance gate.
- Source lock.
- Page-anchor audit.
- Refusal register.
- Cultural-memory register.
- Sibling-boundary register.
- Cross-case checklist.

## Decision

Promote directly to Core under rubric v1.54. Obsidian provenance-hydration
custody creates a reusable control for source matching, hydration dating,
reduction inference, exchange, mobility, and public network claims.
