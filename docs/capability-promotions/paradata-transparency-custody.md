# Paradata Transparency Custody

**Capability id:** `ruins-capability:paradata-transparency-custody`
**Promotion:** Candidate -> Active
**Trigger paper:** `research/publications/panel-seville-paradata-transparency-custody/`
**Date:** 2026-07-07

## Pressure

Virtual reconstructions can look authoritative while hiding the research
sources, assumptions, modelling decisions, uncertainty, and presentation choices
that produced them. RUINS needs a transparency gate for reconstruction process,
not only reconstruction output.

## Activation Rule

Activate this capability when a paper evaluates a digital/virtual
reconstruction, visualization, model, simulation, or public heritage scene whose
claim quality depends on documented sources, metadata, paradata, decision logs,
uncertainty, reuse, and preservation.

## Required Gate

Separate source evidence, metadata, paradata, modelling decisions, uncertainty,
visual output, public communication, sustainability, and reuse.

## Refusals

- A polished visualization is not transparent evidence.
- A model without paradata is not fully auditable.
- Guideline compliance is not the same as historical correctness.
- Public communication does not remove uncertainty.

## Decision

Promote to Active. Queue a concrete reconstruction project as an unlike second
test before Core promotion.
