# Paradata Transparency Checklist

Use this checklist when a digital/virtual reconstruction, model, simulation,
database, restored-state toggle, or public heritage scene is used as evidence.

## Required Separations

| Surface | Required custody question |
|---|---|
| Sources | What material, archival, image, scan, publication, or comparative evidence feeds the model? |
| Metadata | Are object, room, fragment, file, model, version, author, and date records available? |
| Paradata | Are interpretive decisions, alternatives, assumptions, rejected options, and uncertainty logged? |
| Model state | Are actual, restored, hypothetical, missing, and comparative layers distinguishable? |
| Output | What does the visualization, database, toggle, or scene claim to show? |
| Evaluation | Can reviewers test, challenge, cite, or update the reconstruction process? |
| Preservation | Are model, data, documentation, and dependencies sustainable enough for reuse? |
| Communication | Is public access separated from evidential certainty and historical correctness? |

## Refuse Unless Separately Locked

- A polished 3D scene is transparent evidence.
- A restored-state toggle is complete reconstruction.
- Database availability equals paradata sufficiency.
- Guideline compliance proves historical correctness.
- Public navigation recovers original experience.
