# Sealed Cache Representativeness Checklist

Status: Core RUINS checklist
Trigger cases: Dunhuang Library Cave; Cairo Genizah
Capability: `ruins-capability:sealed-cache-representativeness-custody`

Use this checklist when a manuscript, object, document, or mixed cache survives
through storage, hiding, burial, ritual retirement, discard, later removal,
dispersal, cataloguing, digitization, or fragment reassembly.

## Evidence Layers

| Layer | Ask | Blocks |
|---|---|---|
| Deposition practice | Was the material hidden, stored, discarded, retired, cached, or accumulated? | Complete library or archive function. |
| Accumulation span | Was the cache formed at one moment or over long use? | Single event or single institutional purpose. |
| Original order | Is order original, disturbed, reconstructed, catalogued, or unknown? | Full arrangement and original shelf/context claims. |
| Removal history | Who removed the material, when, and under what documentation? | Neutral provenance or ethical closure. |
| Fragment status | Is the unit complete item, fragment, join, bundle, shelfmark, or catalogue record? | Complete corpus and exact textual claims. |
| Collection dispersal | Which holding institution or private collection controls the item now? | Whole-cache access or physical reunion. |
| Digital image/catalogue | Is this image, metadata, transcription, translation, or scholarly edition? | Final reading, complete reunion, or solved corpus. |
| Community/institution memory | Which community, institution, or practice is actually evidenced? | Whole-community or whole-institution history. |
| Public cache memory | Is the phrasing treasure, lost library, sacred trash, or digital reunion? | Original function or representativeness. |

## Required Refusals

- Refuse complete library, archive, or community-history claims from cache
  survival alone.
- Refuse original order unless order evidence is directly source-locked.
- Refuse exact readings, joins, transcriptions, or translations without
  FONTES/MUNDUS custody.
- Refuse digital reunification as physical or interpretive reunification.
- Refuse ownership, acquisition, or repatriation closure from catalogue access.

## Sibling Handoffs

FONTES/MUNDUS own editions, readings, joins, transcriptions, translations, and
language claims. CANON owns named collection, institution, item, and person
identity. MAXIM/LUCIA own broader community and regional history. Legal/policy
systems own ownership, acquisition, return, and ethics. Digital-library systems
own database implementation and image access. CERES owns material conservation.
