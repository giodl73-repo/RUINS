# Shipwreck Context-Loss Checklist

Status: core capability checklist
Owner: ASHLAR / PALIMPSEST / ERRATA
Capability: `ruins-capability:shipwreck-context-loss`
Trigger papers: `panel-uluburun-shipwreck-context-loss`,
`panel-kyrenia-reconstruction-context-loss`

Use this checklist when a shipwreck paper uses recovered cargo, hull remains,
excavation records, conservation, reconstruction, replica, display, catalog, or
dating evidence.

## Required Separations

| Surface | Required RUINS treatment |
|---|---|
| Wreck event | Treat as context break, not as preservation miracle alone. |
| Cargo assemblage | Evidence of material survival and exchange questions, not route, owner, destination, crew, or whole economy by itself. |
| Hull remains | Ancient material evidence mediated by wrecking, recovery, conservation, and reconstruction. |
| Excavation record | Custody layer; exact claims need report, catalogue, or page custody. |
| Conservation | Modern evidence transformation; do not treat stabilized object as unchanged original. |
| Reconstruction / replica | Modern interpretation, experiment, and afterlife; not object-equivalent without explicit status and provenance. |
| Museum / digital catalog | Modern access and reception surface; not original stowage or operating context. |
| Dating / sample work | Method layer; keep probability, model, calibration, and sample limits attached. |
| Route / destination / owner / crew | Refuse unless separately source-locked and qualified. |
| Site/recovery guidance | Refuse operational location, diving, collecting, recovery, or handling guidance. |

## Minimum Artifacts

A mature shipwreck paper needs:

1. `source-lock.md` separating public, publication, catalogue, museum/display,
   reconstruction/replica, and scientific-sample custody.
2. A context-loss gate naming which exact claims are allowed, refused, or
   future-locked.
3. A refusal register or refusal section blocking route, destination, owner,
   crew, cargo-as-economy, display-as-original, and recovery-as-completion
   overclaims.
4. Sibling handoffs for source custody, object identity, route geography,
   broader history, economy/materials, and safety-sensitive recovery work.
5. Page-anchor or publication-custody audit before raising the paper above
   stable working status.

## Core Rule

Do not score a shipwreck paper as publication-ready if it treats recovered
cargo, reconstructed hull, museum display, replica performance, digital catalog,
or scientific dating output as complete original operating context without an
explicit shipwreck context-loss gate.
