# Source Lock: Antikythera Reconstruction

**Owner:** PALIMPSEST
**Status:** first source lock
**Date:** 2026-07-04

This file locks the first source set for the Antikythera working paper. Locking
a source means RUINS may use it for bounded claims. It does not mean any modern
reconstruction is accepted as the ancient object.

## Locked Sources

| ID | Source | Type | Locked use | Limits |
|---|---|---|---|---|
| S1 | Freeth et al., "Decoding the ancient Greek astronomical calculator known as the Antikythera Mechanism" (*Nature*, 2006) | imaging / reconstruction paper | Fragmentary condition, high-resolution X-ray tomography, surface imaging, eclipse/lunar-cycle and back-dial reconstruction framing. | Supports imaging-derived and reconstruction claims, not total certainty. |
| S2 | Freeth et al., "A Model of the Cosmos in the ancient Greek Antikythera Mechanism" (*Scientific Reports*, 2021) | model paper | Front Cosmos model, 82-fragment / 30-gear survival framing, and model-under-constraints language. | A model satisfying evidence is not the ancient object itself. |
| S3 | NYU Faculty Digital Archive Antikythera Mechanism collection | data custody target | PTM files for all 82 presumed fragments, 3D models, early photos, and fragment-level data surfaces. | Dataset custody target; each dataset item needs its own audit before detailed use. |
| S4 | Public summaries / museum and popular coverage | cultural-memory surface | "Ancient computer" framing, wonder, and lost-technology narratives. | Not authority for mechanism layout or gear counts. |

## Locked Values And Claims

| Claim | Locked wording | Source | Grade |

| Fragmentary survival | The mechanism survives as a partial, damaged, corroded fragmentary artifact. | S1 / S2 | Corroborated |

| Data custody | NYU records PTM files for all 82 presumed fragments and related Antikythera data items. | S3 | Custody target |
| Ancient computer language | Public and scholarly sources often call the mechanism an ancient computer or astronomical calculator. | S1 / S2 / S4 | Useful shorthand with definition |

## Paper Use Rules

1. Say "model," "reconstruction," or "source-specific value" when using modern
   gear-train claims.
2. Do not merge preserved gears, CT-revealed features, and inferred gears.
3. Do not treat a replica or 3D reconstruction as the ancient object.
4. Use exact counts only with source labels until page audit is complete.
5. Treat "ancient computer" as shorthand for an analog astronomical calculator,
   not as a modern digital-computing claim.
