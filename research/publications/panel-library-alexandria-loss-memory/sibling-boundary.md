# Sibling Boundary Register: Library of Alexandria Loss and Memory

**Owner:** Sibling Boundary Editor
**Status:** publication-scope boundary register
**Date:** 2026-07-04

RUINS owns the partial-survival and cultural-memory problem. It does not own the
whole history of Alexandria, the full source archive, religious history, or a
canonical identity registry for ancient institutions.

## Boundary Map

| Surface | RUINS role | Handoff | Do not do here |
|---|---|---|---|
| Ptolemaic, Roman, and late antique Alexandria | Mention only where it affects loss layers. | MAXIM | Write a full city or dynasty history. |
| Source editions, PDFs, translations, and scans | Record bounded custody and claim permissions. | FONTES / MUNDUS | Become the source archive. |
| Library, Museum, Serapeum, warehouses | Treat as identity problems with different evidence layers. | CANON | Declare one canonical identity without audit. |
| Alexandria topography | Use only as context for source claims. | PORTO | Build maps or site guides. |
| Religious conflict around the Serapeum | Treat as a loss surface needing source lock. | RITE | Explain late antique religion as a whole. |
| Modern library advocacy and public memory | Treat as afterlife evidence. | RUINS / future reception work | Treat modern symbolism as ancient fact. |

## Expansion Rule

If a section starts requiring maps, institution identity tables, full source
editions, or detailed religious history, it should become a handoff note rather
than remain inside this paper.

## Current Scope Check

The current paper remains a RUINS case study: it separates surviving witness,
source tradition, missing bridges, refusal boundaries, and later memory.
